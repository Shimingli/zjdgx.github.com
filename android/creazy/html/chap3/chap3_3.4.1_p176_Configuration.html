3.4.1 Configuration类简介(P176)

Activity获取系统Configuration对象的方法： Configuration cfg = getResources().getConfigureation();

Android为View组件提供的事件处理方法:

  1. boolean onKeyDown(int keyCode, KeyEvent event);
  2. boolean onKeyLongPress(int keyCode, KeyEvent event);
  3. boolean onKeyShortCut(int keyCode, KeyEvent event);
  4. boolean onKeyUp(int keyCode, KeyEvent event);
  5. boolean onTouchEvent(MotionEvent event);
  6. boolean onTrackballEvent(MotionEvent event): 当用户在该组件上触发轨迹球事件时触发.

基于回调的事件处理机制要通过实现自定义View来实现.

3.3.2 基于回调的事件传播(P172)代码

基于回调的事件处理方法都有一个boolean类型的返回值, 该返回值用于标识该处理方法是否能完全处理该事件:(true: 事件不传播, false: 事件继续传播)

基于监听的事件处理模型的优势:

  1. 基于监听的事件模型分工更明确, 事件源、事件监听两个类分开实现, 具有更好的维护性;
  2. Android的事件处理机制保证基于监听的事件监听器会被优先触发.
